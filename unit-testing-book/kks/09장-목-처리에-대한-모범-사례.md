# 09장 목 처리에 대한 모범 사례

* 비관리 의존성에만 목을 사용
    * 단위 테스트에서는 목 사용 X
* 비관리 의존성과 통신하는 마지막 타입을 목으로 처리
    * 통합 테스트 시 거치는 클래스의 수가 증가함
    * 코드 베이스와 결합도를 낮춰 리팩터링 내성이 더 개선됨
* 목을 스파이로 교체하기
* 테스트당 목이 하나일 필요는 없음
* 호출 횟수 검증하기
    * 목으로 대체한 객체의 호출 여부 뿐만 아니라 정확하게 몇변 호출 하는지도 검증이 필요
* 보유 타입만 목으로 처리하기
    * 서드파티 라이브러리 위에 어댑터를 작성하고 해당 어댑터를 목으로 처리
    * 인메모리 의존성 or 관리 의존성은 추상화 필요 X
